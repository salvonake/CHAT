1. Executive Summary

Poseidon is a production-oriented .NET 8 Windows desktop legal decision support platform. The current architecture is Poseidon-only and is centered on a WPF desktop host, layered domain/application services, local and service-backed AI providers, deterministic provisioning, runtime diagnostics, and an enterprise WiX/Burn installer.

3. Runtime Architecture

Poseidon.Desktop is the primary production UX. Startup initializes configuration and logging, evaluates provisioning state before hosted services start, launches the setup wizard when required, reloads runtime configuration after provisioning, validates model/provider state, and transitions deterministically into Full, Degraded, or Recovery mode.

Poseidon.Api provides JWT-secured legal query and document endpoints. Poseidon.WorkerService handles watched-folder ingestion. Poseidon.Management.Api provides signed management-plane heartbeat and command workflows. Poseidon.ProvisioningCheck validates installer and runtime provisioning contracts.

4. Installer Architecture

WiX MSI plus Burn bundle is the single authoritative enterprise installer path:
- Poseidon.Installer.msi
- Poseidon.Bundle.exe

The installer publishes Desktop and ProvisioningCheck, writes machine config, deploys model assets, emits a schema-versioned model manifest, validates SHA-256 hashes, runs post-install provisioning-check, and fails install success on provisioning failure.

5. Security Architecture

Production config validation is fail-closed. Placeholder JWT secrets, weak secrets, empty management keys, missing DPAPI references, disabled encryption, invalid model hashes, and user attempts to override trusted policy are rejected. Production secrets are referenced through protected storage identifiers, with DPAPI-backed resolution at runtime.

Management-plane requests support signed timestamp/nonce headers and audit logging of auth failures without leaking secret values. Production installer builds require signed artifacts and validated provenance.

6. Build and CI

SDK and package versions are pinned. CI runs repository hygiene validation, unit tests, Qdrant integration tests, optional ONNX integration tests, WiX/Burn installer validation, provisioning-check, manifest tamper checks, silent MSI/Burn install smoke tests, repair smoke tests, and readiness gating.

7. Consolidation Status

Legacy predecessor source, test, and installer paths have been decommissioned in favor of the Poseidon-only build graph and documentation. Repository hygiene policy blocks reintroduction of active legacy references, floating package versions, Docker latest tags, and destructive setup scripts.
